# 白金英雄坛说 - Godot 实现差距审计

> 审计日期：2026-05-31
> 审计范围：以 `docs/00_文档索引.md` 中的新版主文档为准，对照 `godot_project/` 当前 Godot 4 工程。旧版文档和旧原型目录已清理，不作为当前必须实现项。

## 1. 当前实现结论

Godot 版目前是一个可玩的 Alpha 竖切：已经有主菜单、角色创建、连续世界地图、73 个结构化区域、区域探索度、世界地图到局部区域的进入/返回、局部区域相邻转场、局部区域到商铺内景的进入/返回、可互动地标/资源点、驿亭休整与危险野外伏击事件、发现面板、带 NPC/任务标记和点击选择的世界地图面板、基础目的地标记、带路线/耗时/费用/风险提示和基础风险结算的城镇/门派快速旅行、玩家移动、102 个 NPC、对话、教学、商店、背包、带命中/暴击/冷却/状态/敌人风格/掉落/首领阶段的基础战斗、40 个带前置门槛和后续提示的任务、七派各 3 段门派入门任务线、简化修炼、存档读档、江湖传闻/世界事件记录、半俯视 2.5D 阴影/道具投影/景深氛围层、对话世界导演与可选 LLM 后端，以及一批可运行的生成美术资源。

但它距离设计文档中的完整武侠世界还有较大差距。当前主要完成的是“能跑起来的主循环”，文档中的区域系统、剧情系统、复杂战斗、经济声望、NPC 关系、天气氛围、完整 UI 和大规模美术资产，大多还没有在 Godot 中落地。

## 4. 主要未实现项

### 4.1 世界观与势力

- 七大门派已有名称、基础技能表、部分 NPC 分布和各 3 段入门任务线；门派历史、组织结构、门规、声望奖励、内部矛盾没有系统化实现。
- 中立势力没有实现：天机阁、铁血盟、药王谷、暗影司、血月教、龙门客栈。
- 五大城池的文化、经济差异、特色 NPC、隐藏区域、动态事件没有对应系统。
- 已有轻量江湖传闻/世界事件记录；还没有季节事件、随机世界事件、势力冲突事件和玩家选择造成的复杂世界变化。

### 4.2 世界地图

- 当前世界层仍是压缩连续大地图，不是文档要求的 73 个完整独立区域场景。

## 5. 优先级建议

## 6. 近期最合理的下一步

建议下一轮不要直接继续堆 NPC 或美术，而是把已接入的“世界地图 -> 局部区域 -> 商铺内景”扩展成数据驱动区域系统。原因是地图、NPC、任务、奇遇、快速旅行、天气、资源点都依赖区域 ID；没有这个底座，后续内容会继续散落在坐标和硬编码里。
