# 白金英雄坛说 - Godot 实现差距审计

> 审计日期：2026-05-31
> 审计范围：以 `docs/00_文档索引.md` 中的新版主文档为准，对照 `godot_project/` 当前 Godot 4 工程。旧版文档和旧原型目录已清理，不作为当前必须实现项。

## 1. 当前实现结论

Godot 版目前是一个可玩的 Alpha 竖切：已经有主菜单、角色创建、连续世界地图、73 个结构化区域、区域探索度、世界地图到局部区域的进入/返回、局部区域相邻转场、局部区域到商铺内景的进入/返回、可互动地标/资源点、发现面板、带 NPC/任务标记和点击选择的世界地图面板、基础目的地标记、城镇/门派快速旅行、玩家移动、99 个 NPC、对话、教学、商店、背包、简化战斗、10 个任务、简化修炼、存档读档，以及一批可运行的生成美术资源。

但它距离设计文档中的完整武侠世界还有较大差距。当前主要完成的是“能跑起来的主循环”，文档中的区域系统、剧情系统、复杂战斗、经济声望、NPC 关系、天气氛围、完整 UI 和大规模美术资产，大多还没有在 Godot 中落地。

## 4. 主要未实现项

### 4.1 世界观与势力

- 七大门派只有名称、基础技能表和部分 NPC 分布；门派历史、组织结构、门规、入门试炼、声望奖励、内部矛盾没有系统化实现。
- 中立势力没有实现：天机阁、铁血盟、药王谷、暗影司、血月教、龙门客栈。
- 五大城池的文化、经济差异、特色 NPC、隐藏区域、动态事件没有对应系统。
- 世界动态事件没有实现：季节事件、随机世界事件、势力冲突事件、玩家选择造成的世界变化。

### 4.2 世界地图

- 当前世界层仍是压缩连续大地图，不是文档要求的 73 个完整独立区域场景。
- 已有区域 ID 数据结构、进入识别、局部区域模板、相邻区域路牌转场、可互动地标/资源点和独立商铺内景；局部地图已按地貌和区域类型做第一轮差异化，但还没有为 73 个区域逐一定制的正式地图布局、隐藏点和事件层。
- 已有基础探索度记录、一次性地标奖励、每日刷新资源点和发现面板；还没有隐藏区域发现、区域称号、区域 Boss 解锁。
- 已有发现区域后的城镇/门派基础快速旅行和旅行时间推进；还没有传送阵、通行证限制、驿站费用和野外旅行风险。
- 已有大地图面板、HUD 小地图、NPC/任务目标标记和目的地标记；还没有筛选和路线规划。
- 五大城池、主要城镇、七大门派已有第一版局部地图、相邻区域入口和商铺/功能入口模板；还没有按城市文化差异做完整街区、府衙、武馆、黑市、门派内部建筑和专属事件。

### 4.3 NPC 系统

- `03_npc_design.md` 中 31 个明确命名的核心/门派 NPC 已落地 10 个：苏梦瑶、陈天行、赵无极、玄机子、花如玉、烈火、蛇王、太极真人、冰魄、逍遥子；其余核心 NPC 仍未实现。
- 已有 NPC 性格字段展示、基础好感度和交互记忆保存；还没有完整 8 维性格模型、人物弧光、日常作息和关系网络传播。
- 没有 NPC 对玩家行为的长期反馈，例如救助、背叛、亲友死亡、声望变化后的台词变化。
- 城镇 NPC 模板已有少量抽象外观，但没有模板化作息、服务类型、可复用任务池。
- 敌人只有静态地图敌人，没有文档中的山贼、流寇、邪教徒、野兽、精怪、朝廷鹰犬体系。

### 4.4 物品与装备

- 当前 Godot 只有 22 个物品，文档目标是 400+ 物品。
- 物品分类不完整：没有材料、秘籍、任务物品、宝物、特殊货币、毒药体系。
- 没有稀有度、鉴定、耐久、修理、强化、镶嵌、套装、打造、炼丹、采集制作。
- 装备槽只有武器和防具，文档要求头部、身体、手部、脚部、饰品等多槽位。
- 没有门派契合度加成和传说武器独特技能。
- 没有出售物品、交易、掉落表、隐藏宝箱。

### 4.5 战斗

- 当前战斗是简化 1v1 回合制，没有完整八阶段状态机。
- 文档中的 30+ 武学招式只保留了技能名，未实现招式效果、MP 消耗、冷却、伤害倍率、状态附加。
- 没有五行/门派属性克制、暴击、命中、闪避、格挡、连击、反击。
- 没有中毒、灼烧、冰冻、眩晕、流血、内伤等状态效果。
- 没有敌人 AI 决策树、血量分档、门派 AI、协作战术、Boss 多阶段机制。
- 没有世界 Boss、区域随机遇敌、敌人生成规则、等级缩放、掉落奖励。
- 没有阵型、前后排、地形战斗加成。
- 战斗 UI 没有立绘、伤害数字、状态图标、技能子菜单、战斗动画和水墨特效。

### 4.6 修炼

- 当前只是 `learn_skill` 和 `practice_skill`，没有独立修炼系统类。
- 没有八大内功的属性曲线、修炼条件、特殊效果。
- 没有五大轻功、移动速度/闪避加成公式。
- 没有经脉系统、八脉打通、突破路线、突破消耗。
- 没有大境界突破、天象变化、悟道事件、走火入魔风险。
- 没有师徒传承、秘籍传承、顿悟传承。
- 没有修炼与装备、战斗属性的完整联动。
- 技能面板不是技能树，只是列表升级。

### 4.7 经济与声望

- 当前只有单一银两，没有文/两/金三种货币单位和兑换关系。
- 商店已有客栈、药铺、铁匠铺、布庄、市集、茶肆等第一批类型入口和内景，但商品、折扣、出售、修理、黑市、地区差价仍很浅。
- 没有动态市场、供需、季节价格、事件价格、地区差价套利。
- 没有拍卖行、黑市、钱庄、存款利息、贷款、钱庄信誉。
- 没有门派声望、城市声望、NPC 好感、名望等级。
- 没有正邪值影响系统；当前只有 `daode` 数值奖励，缺少实际规则影响。

### 4.8 剧情与奇遇

- 当前已有 10 个 Alpha 任务和一条主线开端，但还不是文档中的五章主线。
- 没有八阶段剧情架构、主线节点图、剧情选择、分支后果。
- 没有八种结局和 New Game+。
- 没有 50+ 支线任务、任务失败后果、任务时间限制。
- 没有八大随机奇遇事件和权重触发机制。
- 门派任务线只做了八卦入门一条很短的任务，没有花间、红莲、那迦、太极、雪山、逍遥完整任务线。
- 没有 NPC 关系剧情、NPC 死亡剧情、世界状态不可逆选择。

### 4.9 天气与氛围

- 当前已有时间和天气文字、昼夜光照色调、区域地貌色调、危险区域压暗、细雨/飞雪/薄雾/多云视觉层，以及水乡、城镇、山地、花林、荒漠、温泉、雪山和门派气场等程序化动态氛围。
- 还没有 9 种完整天气参数、季节效果和真实光照阴影。
- 没有全局光照、阴影、反射、雾效。
- 已有程序化雨雪雾、落叶、沙尘、飞花、雪光和门派气场视觉表现；还没有正式 GPU 粒子资源。
- 没有环境音效、3D 音效定位、动态音乐分层。
- 没有 `AtmosphereManager` 或对应 Godot 节点。

### 4.10 UI

- 当前 UI 是可用的程序化面板，但不符合 `11_UI设计.md` 中完整的水墨国风规范。
- 主菜单没有完整背景、开场动画和水墨过渡。
- 角色创建没有门派说明、随机姓名、角色立绘预览。
- HUD 已有区域信息和常驻小地图，但没有技能栏、任务追踪详情、丰富状态图标和路线指引。
- 背包已有物品图标和详情预览；仍没有 6x5 网格、分类标签、拖拽、Tooltip、品质边框、自动整理。
- 技能面板已有武学图标和详情预览；仍没有门派技能树、节点解锁、技能动画。
- 对话已有 NPC 头像；仍缺正式立绘、打字机效果和历史对话。
- 商店已有商品图标和详情预览；仍没有排序、折扣、出售、交易确认。
- 战斗 UI 已显示敌人头像，并会按已学攻击类武学生成招式按钮；仍没有完整行动菜单、伤害浮字、状态图标、日志滚动。
- 没有锻造/炼丹界面、设置界面、存档管理界面；世界地图界面已有基础版和点击定位/快速旅行，但仍缺筛选、小地图和路线规划。
- 没有响应式 UI 缩放、键位自定义、加载界面。

### 4.11 美术资产

- 当前新增第一批 Godot 可直接使用的生成资源：20 张地图瓦片、99 张 NPC 地图 sprite、41 张参考级 NPC archetype、99 张 NPC 对话头像、16 张玩家门派/性别 sprite、45 张角色拆件 PNG、22 张物品图标、41 张武学图标、73 张区域场景背景、8 张 UI 资源，并有 `assets/generated_art_manifest.json` 记录清单。
- 当前保留并接入 41 个高精度水墨 Q 版参考级地图 NPC archetype、99 个组件化地图 NPC sprite 和 99 个对话头像；旧 NPC 图集/切片源已清理，`12_美术资产Prompt.md` 的长期目标仍是 900+ 总资产。
- 当前大地图 NPC 已从纯程序化绘制改为优先使用参考级透明 PNG sprite，未覆盖时回退到组件化生成 sprite，并完成角色缩放、姓名标签显隐、出生点和 NPC 分布修正；对话面板已接入头像；背包/商店已接入物品图标；修炼/战斗已接入武学图标；世界地图面板已接入区域背景和地图标记。这仍只是可运行资产层优化，不等同于正式高精立绘、全套战斗姿态和动画完成。
- 缺失正式角色立绘、战斗姿态、门派角色完整变体；已有头像仍是统一生成风格的半身占位。
- 已生成五大城池、十六小镇、四十五野外、七大门派对应的 73 张区域场景背景占位图，并接入世界地图区域详情；局部区域和商铺内景已可进入，并已有按城镇/门派/地貌区分的程序化布局和 2.5D 道具层，但仍不是最终高精场景图或正式 TileSet。
- 已有第一批地图瓦片 PNG；仍缺正式 TileSet 资源配置、自动地形拼接、建筑/道路/河流/山体/树木的多变体素材。

### 4.12 技术架构

- Godot 工程还没有把背包、任务、修炼、经济、声望等规则拆成独立系统节点。
- 缺少独立 `inventory_system.gd`、`quest_system.gd`、`cultivation_system.gd`、`economy_system.gd`、`reputation_system.gd`、`story_system.gd`、`atmosphere_system.gd`。
- 当前很多规则集中在 `GameState`，后续功能变多后需要拆分。
- 地图还不是 TileMapLayer/TileSet，后续正式美术接入会受限。
- 没有自动化测试、数据校验、导出配置、性能基准。
- `10_技术架构.md` 已切换为 Godot 架构说明，后续需要随系统拆分持续更新。

## 5. 优先级建议

### P0：先把当前 Alpha 变成稳定可扩展骨架

1. 把任务、背包、修炼、经济从 `GameState` 拆为独立系统。
2. 给 Godot 数据加校验脚本：NPC 坐标可通行、商品引用存在、任务目标存在、技能 ID 存在。
3. 增加世界地图数据层：区域 ID、区域类型、所属城市、探索状态、进入触发。
4. 补一份 Godot 专用 backlog，让 `13_Godot重构设计.md` 和本审计文档成为实现依据。

### P1：补“真正武侠世界”的关键体验

1. 在已完成的世界到局部区域转入转出基础上，为 5 大城、16 镇、45 野外、7 门派分批补定制布局、区域事件、资源点和隐藏入口。
2. 按 `03_npc_design.md` 落地核心 NPC 第一批：苏梦瑶、陈天行、玄机子、太极真人、逍遥子等。
3. 做门派入门和门派任务线第一轮，每个门派至少 3 个任务。
4. 扩展战斗招式系统：技能数据、招式效果、状态效果、敌人 AI。
5. 世界地图面板、HUD 小地图、任务标记、NPC 标记、点击定位和基础快速旅行已做基础版；后续需要加筛选、路线规划和旅行限制。

### P2：扩展系统深度

1. 物品扩展到 100+，先做稀有度、装备槽、掉落表、出售、材料。
2. 经济扩展到商店类型、折扣、声望、正邪值影响。
3. 修炼扩展到内功、轻功、经脉和突破。
4. 剧情扩展到第一章完整主线和至少 10 条支线。
5. 天气和氛围已从文字推进到画面色调与程序化动效；后续继续补音效、季节、真实光照和正式粒子资源。

### P3：完整内容量与美术资产

1. 生成正式 TileSet 和区域场景素材。
2. 继续补全 UI 皮肤、武器装备图标、技能特效，并把已生成图标接入背包/商店。
3. 补齐五城、七派、十六镇、四十五野外的场景差异。
4. 实现多结局、New Game+、世界事件连锁。

## 6. 近期最合理的下一步

建议下一轮不要直接继续堆 NPC 或美术，而是把已接入的“世界地图 -> 局部区域 -> 商铺内景”扩展成数据驱动区域系统。原因是地图、NPC、任务、奇遇、快速旅行、天气、资源点都依赖区域 ID；没有这个底座，后续内容会继续散落在坐标和硬编码里。

建议具体拆成三步：

1. 已完成 `data/regions.json`，把 5 城、16 镇、45 野外、7 门派全部登记为结构化区域。
2. 已完成 `WorldMap` 当前区域识别，并在 HUD 显示区域名、探索度、危险等级。
3. 已完成基础地图面板，显示已发现区域、区域说明、NPC/任务标记，并支持点击定位、目的地标记和城镇/门派快速旅行。
4. 已完成第一版区域进入、相邻区域路牌转场、可互动地标/资源点和商铺内景；下一步应把局部地图布局、商铺清单、隐藏区域入口、路线规划和旅行限制改为数据驱动。
