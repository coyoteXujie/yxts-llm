> **文档版本**: v3.0（深度优化版） | **最后更新**: 2026-05-19 | **对应代码**: window.py / hud_renderer.py / combat_renderer.py  
> **引擎**: Arcade (Python)

## 一、UI设计概述

### 1.1 设计哲学

《白金英雄坛说》的UI设计遵循"沉浸式武侠体验"的核心原则。我们不希望UI成为玩家与游戏世界之间的屏障，而是希望UI本身也成为武侠世界的一部分——如同古代侠客手中的折扇、腰间的玉佩，既是工具，也是身份的象征。

**设计三原则：**

1. **意境优先**：每个UI元素都应该传达武侠意境，而非单纯的功能指示
2. **留白美学**：遵循中国传统绘画的留白理念，避免信息过载
3. **动态韵律**：UI不是静态的，而是随着游戏节奏呼吸、流动

### 1.2 视觉语言体系

**水墨质感系统：**

游戏采用五层水墨渲染体系：
- **浓墨层**：用于重要信息和强调元素，透明度90-100%
- **重墨层**：用于面板背景和主要容器，透明度70-85%
- **淡墨层**：用于次要信息和分隔线，透明度40-60%
- **清墨层**：用于背景纹理和装饰，透明度15-30%
- **飞白层**：用于高光和特殊效果，透明度10-20%

**金色点缀系统：**

金色在游戏中代表"尊贵"和"重要"，使用场景包括：
- 边框和分割线
- 标题和重要文字
- 选中状态和高亮
- 稀有物品和高级技能

金色采用动态渐变，从 `#c9a96e`（古铜金）到 `#f0c040`（亮金），营造金属光泽感。

**朱砂强调系统：**

红色用于警示、伤害、重要提示，色值为 `#c0392b`（深朱砂）和 `#e74c3c`（亮朱砂）。

### 1.3 交互设计原则

**反馈即时性：**
- 所有交互操作必须在100ms内产生视觉反馈
- 悬停效果延迟不超过50ms
- 点击效果即时触发

**操作一致性：**
- 相同类型的操作在不同界面中保持一致
- 快捷键映射遵循行业惯例（I=背包，K=技能，M=地图）
- 鼠标和键盘操作可以互相替代

**学习曲线：**
- 新手引导通过UI动画自然融入
- 复杂功能分阶段解锁，避免信息过载
- 所有功能都有悬停提示

## 二、UI风格定义

### 2.1 水墨国风设计语言

#### 2.1.1 墨色层次系统

游戏中的"墨"不是单一的颜色，而是一个完整的色彩体系：

```
浓墨 (#0a0a0f) —— 用于最重要的文字和图标
    ↓
重墨 (#1a1a2e) —— 用于面板背景
    ↓
中墨 (#2a2a3e) —— 用于次级面板
    ↓
淡墨 (#3a3a4e) —— 用于分隔线和边框
    ↓
清墨 (#4a4a5e) —— 用于装饰性元素
```

每种墨色都有对应的透明度变体，用于不同的叠加场景。

#### 2.1.2 笔触纹理系统

游戏使用程序生成的笔触纹理来模拟真实的水墨效果：

**干笔纹理**：用于边框和分割线，模拟毛笔干涩时的飞白效果
**湿笔纹理**：用于按钮和选中状态，模拟墨汁饱满时的晕染效果
**泼墨纹理**：用于特殊效果和动画，模拟泼墨山水的随机美感
**皴法纹理**：用于背景装饰，模拟山水画中的皴擦技法

#### 2.1.3 宣纸质感模拟

游戏背景模拟宣纸的质感：
- 基底颜色：`#12121a`（如墨夜深沉）
- 纹理叠加：使用 `texture_noise_intensity: 0.05` 添加细微噪点
- 边缘晕染：面板边缘有轻微的墨色渐变

### 2.2 色彩心理学应用

#### 2.2.1 为什么选择这些颜色

**主背景色 `#12121a`：**
- 心理学依据：深色背景减少视觉疲劳，适合长时间游戏
- 文化含义：如深夜、如墨池，营造神秘、沉稳的武侠氛围
- 技术考量：深色背景使彩色元素更加突出

**金色 `#c9a96e`：**
- 心理学依据：金色激发尊贵感和成就感
- 文化含义：中国古代皇家专用色，象征权力和财富
- 应用场景：边框、标题、重要按钮、稀有物品

**红色 `#c0392b`：**
- 心理学依据：红色是最醒目的颜色，用于警示和强调
- 文化含义：中国传统的喜庆色，也代表危险和战斗
- 应用场景：HP条低血量、伤害数字、警告提示

**绿色 `#2ecc71`：**
- 心理学依据：绿色代表生命、恢复、安全
- 文化含义：竹林、青山，武侠世界的自然元素
- 应用场景：HP条高血量、治疗数字、正面状态

**蓝色 `#3498db`：**
- 心理学依据：蓝色代表冷静、智慧、内力
- 文化含义：水、天、道家修炼
- 应用场景：MP条、技能图标、水系武学

#### 2.2.2 色彩搭配规则

**主色+辅色+点缀色原则：**
- 每个界面最多使用3种主要颜色
- 主色占60%，辅色占30%，点缀色占10%
- 避免使用互补色直接对比（如红绿并置）

**门派色彩体系：**

| 门派 | 主色 | 辅色 | 色彩含义 |
|------|------|------|----------|
| 八卦门 | `#4a3b6b` 紫黑 | `#c9a96e` 金 | 玄奥、变化 |
| 花间派 | `#c9708a` 粉红 | `#f0c040` 金 | 柔美、灵动 |
| 红莲教 | `#8b2020` 深红 | `#f0c040` 金 | 刚烈、炽热 |
| 那迦派 | `#5b3a8c` 暗紫 | `#c9a96e` 金 | 神秘、异域 |
| 太极门 | `#8a9aaa` 银灰 | `#c9a96e` 金 | 平衡、中正 |
| 雪山派 | `#5a8aaa` 冰蓝 | `#c9a96e` 金 | 清冷、高洁 |
| 逍遥派 | `#4a8a5a` 翠绿 | `#c9a96e` 金 | 自然、自在 |

### 2.3 字体选择与设计

#### 2.3.1 字体体系

**标题字体：思源宋体 (Noto Serif SC) Bold**
- 选择理由：宋体的笔画粗细变化模拟毛笔书写，符合武侠主题
- 使用场景：游戏标题、面板标题、重要按钮
- 字号范围：24px - 72px
- 字重：Bold (700)

**正文字体：思源黑体 (Noto Sans SC) Regular/Medium**
- 选择理由：黑体清晰易读，适合大段文字
- 使用场景：描述文字、属性数值、对话内容
- 字号范围：12px - 18px
- 字重：Regular (400) / Medium (500)

**装饰字体：方正清刻本悦宋**
- 选择理由：仿古刻本风格，增强历史感
- 使用场景：游戏Logo、章节标题、特殊装饰
- 字号范围：48px - 72px

#### 2.3.2 字体渲染优化

**动态字体加载：**
- 使用Arcade的字体缓存机制
- 中文字符子集化，仅加载游戏中实际使用的字符
- 字体纹理动态生成，避免内存浪费

**字体特效：**
- 标题文字添加微弱的发光效果（Glow）
- 重要数字使用描边增强可读性
- 选中文字添加金色下划线

### 2.4 留白与呼吸感

#### 2.4.1 间距系统

游戏采用8px基准的间距系统：

```
xs: 4px   —— 图标与文字间距
sm: 8px   —— 紧凑元素间距
md: 16px  —— 标准元素间距
lg: 24px  —— 宽松元素间距
xl: 32px  —— 面板内边距
2xl: 48px —— 面板间距
```

#### 2.4.2 留白原则

**内容区域留白：**
- 面板内边距至少16px
- 相关内容组之间留白24px
- 不相关内容之间留白32px

**视觉留白：**
- 避免将元素填满整个屏幕
- 重要元素周围留出"呼吸空间"
- 使用透明度变化创造虚拟留白

#### 2.4.3 视觉层次

通过留白建立视觉层次：
1. **一级元素**：最大、最醒目，周围留白最多
2. **二级元素**：中等大小，周围留白适中
3. **三级元素**：最小，可以紧密排列

```python
def get_hp_color(ratio):
    """
    HP条颜色根据血量比例动态变化
    ratio: 0.0 - 1.0
    """
    if ratio > 0.5:
        # 高血量：绿色 (60, 200, 60)
        return (60, 200, 60)
    elif ratio > 0.25:
        # 中血量：黄绿色 (200, 180, 40)
        t = (ratio - 0.25) / 0.25  # 0-1插值
        return interpolate_color((200, 180, 40), (60, 200, 60), t)
    else:
        # 低血量：红色 (220, 50, 50)，带闪烁
        return (220, 50, 50)

# HP条渐变绘制
def _draw_bar(x, y, w, h, current, maximum, c_high, c_mid, c_low, bg, border, label):
    ratio = current / maximum if maximum > 0 else 0
    fw = max(1, int(w * ratio))  # 填充宽度
    
    # 背景
    draw_gradient_rect(x, y, w, h, lighten(bg, 5), darken(bg, 5), 3)
    
    # 填充
    c = c_high if ratio > 0.5 else (c_mid if ratio > 0.25 else c_low)
    draw_gradient_rect(x, y, fw, h, lighten(c, 25), darken(c, 10), 4)
    
    # 高光层
    arcade.draw_rect_filled(arcade.LBWH(x, y + h - 4, fw, 4), alpha(lighten(c, 35), 45))
    arcade.draw_rect_filled(arcade.LBWH(x, y + h - 2, fw, 2), alpha(lighten(c, 50), 25))
    
    # 边框
    arcade.draw_rect_outline(arcade.LBWH(x, y, w, h), border, 1)
    
    # 文字
    draw_text(f"{label} {current}/{maximum}", x + w / 2, y + h / 2, (255, 255, 255), 10, "center", "center")
```

### 6.5 整理背包自动排序算法

**排序规则（可配置）：**

```python
SORT_RULES = {
    "default": ["type", "quality", "level"],  # 默认：类型→品质→等级
    "quality": ["quality", "level", "type"],  # 品质优先
    "level": ["level", "quality", "type"],   # 等级优先
    "name": ["name"],                          # 名称
}

def sort_inventory(items, rule="default"):
    """
    整理背包排序算法
    """
    priority = SORT_RULES[rule]
    
    # 类型优先级
    type_priority = {
        "weapon": 1,
        "armor": 2,
        "accessory": 3,
        "consumable": 4,
        "material": 5,
        "quest": 6,
    }
    
    # 品质优先级
    quality_priority = {
        "legendary": 1,
        "epic": 2,
        "rare": 3,
        "uncommon": 4,
        "common": 5,
    }
    
    def sort_key(item):
        keys = []
        for p in priority:
            if p == "type":
                keys.append(type_priority.get(item.type, 99))
            elif p == "quality":
                keys.append(quality_priority.get(item.quality, 99))
            elif p == "level":
                keys.append(-item.level)  # 高等级在前
            elif p == "name":
                keys.append(item.name)
        return keys
    
    return sorted(items, key=sort_key)
```

**整理动画：**
- 物品"飞"到目标位置
- 持续时间：0.3秒
- 缓动：EaseOutQuad
- 音效：整理完成音效

## 十五、响应式设计（新增）

### 15.1 不同分辨率适配

**基准分辨率：** 1440×900

**缩放公式：**
```python
def calculate_scale(width, height):
    """计算UI缩放比例"""
    base_width = 1440
    base_height = 900
    
    # 计算宽高比
    scale_x = width / base_width
    scale_y = height / base_height
    
    # 取较小值，保持比例
    scale = min(scale_x, scale_y)
    
    # 限制缩放范围
    scale = max(0.7, min(scale, 1.5))
    
    # 取整到0.1
    return round(scale * 10) / 10
```

**适配策略：**

| 分辨率 | 缩放 | 布局调整 |
|--------|------|----------|
| 1920×1080 | 1.2x | 标准布局 |
| 1440×900 | 1.0x | 标准布局 |
| 1366×768 | 0.9x | 紧凑布局 |
| 1280×720 | 0.8x | 简化装饰 |
| 1024×768 | 0.7x | 最小布局 |

### 15.2 UI缩放选项

**缩放级别：**
- 自动（根据分辨率）
- 小（0.8x）
- 中（1.0x）
- 大（1.2x）
- 超大（1.4x）

**缩放影响：**
- 所有UI元素尺寸
- 字体大小
- 间距
- 图标尺寸

### 15.3 宽高比适配

| 比例 | 适配策略 |
