# 经济与声望系统

> 当前 Godot 对应：`godot_project/scripts/ui/shop_panel.gd` / `godot_project/scripts/ui/dialogue_panel.gd` / `godot_project/scripts/autoload/game_state.gd`
>
> 本文档深度解析"白金英雄坛说"的经济与声望系统，涵盖货币体系、商店系统、拍卖行、黑市、钱庄、声望等级、正邪阵营等核心机制。本系统模拟了真实的江湖经济生态，为玩家提供沉浸式的商业与社交体验。

---

## 一、经济系统概述

### 1.1 江湖经济哲学

在中国传统武侠世界中，经济不仅仅是金钱的交易，更是人情世故的体现。"白金英雄坛说"的经济系统深刻体现了这一哲学：

**义利之辨**

江湖中人讲究"义"字当头，但"利"亦不可忽视。经济系统鼓励玩家通过正当途径获取财富，同时也提供了黑市等特殊渠道，让玩家面临道德选择。

**供需之道**

物价波动系统模拟了真实的市场经济，供需关系决定价格。玩家可以通过低买高卖赚取差价，体验商人之道。

**信誉之本**

声望系统与经济系统紧密相连，良好的声望可以带来折扣优惠，而恶劣的声望则会被商家加价甚至拒绝服务。

### 1.2 系统架构总览

经济系统由七大核心模块构成：

| 模块 | 核心功能 | 玩家体验 |
|------|----------|----------|
| **货币体系** | 铜钱、银两、金锭的兑换与管理 | 财富积累与消费 |
| **商店系统** | 九种商店类型，各类商品买卖 | 日常购物与装备更新 |
| **拍卖行** | 稀有物品竞拍，价高者得 | 获取稀有物品的途径 |
| **黑市系统** | 违禁品交易，高风险高回报 | 特殊商品的获取 |
| **钱庄系统** | 存取款、利息计算、贷款 | 财富管理 |
| **动态市场** | 供需影响价格，季节性波动 | 商业投机 |
| **声望系统** | 门派声望、正邪阵营、NPC关系 | 社交与道德选择 |

## 二、货币体系详解

### 2.1 三种货币单位

游戏采用三级货币体系，模拟古代中国的货币制度：

| 货币单位 | 换算关系 | 主要用途 | 获取难度 |
|----------|----------|----------|----------|
| **铜钱** | 1银两 = 1000铜钱 | 日常小额消费 | 极易 |
| **银两** | 基础货币单位 | 主要交易货币 | 中等 |
| **金锭** | 1金锭 = 100银两 | 大额交易、储蓄 | 较难 |

### 2.2 货币兑换关系

```
1 金锭 = 100 银两 = 100,000 铜钱
1 银两 = 1000 铜钱
```

**兑换手续费**：
- 铜钱换银两：无手续费
- 银两换金锭：收取1%手续费
- 金锭换银两：收取2%手续费

### 2.3 货币获取途径（8种+）

| 获取途径 | 获得数量 | 难度 | 风险 | 说明 |
|----------|----------|------|------|------|
| **完成任务** | 50-500银两 | 低 | 无 | 主线、支线任务奖励 |
| **战斗掉落** | 10-100银两 | 中 | 中 | 击败敌人后掉落 |
| **出售物品** | 视物品而定 | 低 | 无 | 将不需要的物品卖给商店 |
| **采集资源** | 5-50银两 | 低 | 低 | 采集草药、矿石等 |
| **拍卖行** | 视竞拍而定 | 中 | 低 | 出售稀有物品 |
| **黑市交易** | 高额利润 | 高 | 高 | 买卖违禁品 |
| **钱庄利息** | 本金的0.1%/天 | 无 | 无 | 存款获得利息 |
| **投资商铺** | 视经营而定 | 中 | 中 | 投资商铺获得分红 |
| **赌博** | 翻倍或归零 | 低 | 极高 | 在赌场赌博 |
| **偷窃** | 随机金额 | 低 | 高 | 偷窃NPC或商店 |

### 2.4 货币消耗优先级

系统会自动按照以下优先级消耗货币：

1. **铜钱**：优先消耗，用于小额支付
2. **银两**：铜钱不足时使用
3. **金锭**：最后使用，用于大额支付

**自动兑换机制**：
- 当银两不足但金锭充足时，系统会自动将金锭兑换为银两
- 兑换时会扣除相应的手续费

## 四、动态市场系统

### 4.1 供需影响公式详解

物价波动基于供需系数实时计算：

```python
# 供需比计算
supply_demand_ratio = supply / max(0.1, demand)

# 价格系数计算
price_factor = 1.0 / supply_demand_ratio

# 价格系数限制在0.5-2.0之间
price_factor = max(0.5, min(2.0, price_factor))

# 当前价格计算
current_price = int(base_price * price_factor)
```

**价格波动范围**：

| 情况 | 供需比 | 价格系数 | 实际价格 |
|------|--------|----------|----------|
| 供大于求 | > 2.0 | 0.5（下限） | 基础价 x 0.5 |
| 供需平衡 | 1.0 | 1.0 | 基础价 x 1.0 |
| 求大于供 | < 0.5 | 2.0（上限） | 基础价 x 2.0 |

## 五、拍卖行系统

### 5.1 拍卖流程详解

**拍卖流程**：

1. **物品上架**：卖家将物品交给拍卖行，设置起拍价和拍卖时长
2. **展示期**：物品在拍卖行展示，玩家可以查看物品信息
3. **竞拍期**：玩家可以出价竞拍，出价必须高于当前最高价
4. **结算期**：拍卖结束后，最高价者获得物品，卖家获得金币

**拍卖时长选项**：

| 时长 | 费用 | 适用物品 |
|------|------|----------|
| 1小时 | 1% | 普通物品 |
| 4小时 | 2% | 优秀物品 |
| 12小时 | 3% | 稀有物品 |
| 24小时 | 5% | 史诗物品 |
| 72小时 | 8% | 传说物品 |

### 5.2 竞价策略

**常见竞价策略**：

| 策略名称 | 策略描述 | 适用情况 |
|----------|----------|----------|
| **狙击策略** | 在拍卖结束前最后一刻出价 | 竞争激烈的热门物品 |
| **逐步加价** | 每次加价最小额度 | 竞争不激烈的物品 |
| **心理价位** | 设定最高出价，不超过即停止 | 理性消费 |
| **威慑出价** | 一次性出高价吓退竞争者 | 资金充足时 |
| **观望策略** | 先观察，最后决定是否出价 | 不确定物品价值时 |

### 5.3 拍卖税

拍卖成功后，系统会收取一定比例的税费：

| 税费类型 | 税率 | 说明 |
|----------|------|------|
| **成交税** | 5% | 从成交金额中扣除 |
| **上架费** | 1-8% | 根据拍卖时长收取 |
| **流拍费** | 2% | 物品流拍时收取 |

**税费计算示例**：
- 物品成交价：1000银两
- 成交税：1000 x 5% = 50银两
- 上架费（24小时）：1000 x 5% = 50银两
- 卖家实际获得：1000 - 50 - 50 = 900银两

### 5.4 稀有物品出现规律

稀有物品的出现有一定规律：

| 稀有度 | 出现频率 | 最佳拍卖时间 |
|--------|----------|--------------|
| 普通 | 每天 | 任何时间 |
| 优秀 | 每天 | 任何时间 |
| 稀有 | 每周2-3次 | 周末 |
| 史诗 | 每月1-2次 | 节假日 |
| 传说 | 每季度1次 | 特殊活动期间 |

**提高获得稀有物品的方法**：
- 提升名望等级
- 与拍卖行NPC建立良好关系
- 参与特殊活动
- 完成特定任务

## 七、钱庄系统

### 7.1 存款利息计算

钱庄提供存款服务，存款可以获得利息。

**利息计算公式**：

```
利息 = 存款余额 x 日利率 x 存款天数
```

**日利率**：0.1%（0.001）

**复利计算**：

```
最终金额 = 本金 x (1 + 日利率)^天数
```

**利息计算示例**：

| 存款金额 | 存款天数 | 单利利息 | 复利利息 |
|----------|----------|----------|----------|
| 1000银两 | 30天 | 30银两 | 30.42银两 |
| 1000银两 | 90天 | 90银两 | 93.54银两 |
| 1000银两 | 365天 | 365银两 | 440.03银两 |
| 10000银两 | 30天 | 300银两 | 304.17银两 |
| 10000银两 | 365天 | 3650银两 | 4400.32银两 |

### 7.2 贷款系统

钱庄提供贷款服务，但需要满足一定条件。

**贷款条件**：

| 条件类型 | 要求 | 说明 |
|----------|------|------|
| **抵押** | 价值贷款金额150%的物品 | 降低钱庄风险 |
| **信用评估** | 信用分达到60以上 | 根据还款记录评估 |
| **担保人** | 有声望的NPC担保 | 降低贷款门槛 |

**贷款利率**：

| 贷款类型 | 日利率 | 说明 |
|----------|--------|------|
| **抵押贷款** | 0.2% | 需要抵押物 |
| **信用贷款** | 0.5% | 需要良好信用 |
| **高利贷** | 1.0% | 无抵押，风险高 |

**还款期限**：
- 短期：7天
- 中期：30天
- 长期：90天

**逾期后果**：
- 第1-3天：每日增加1%滞纳金
- 第4-7天：信用分下降，被追债
- 第8天+：抵押物被没收，被通缉

### 7.3 钱庄信誉

不同钱庄的可靠性不同：

| 钱庄名称 | 信誉等级 | 利率 | 特点 |
|----------|----------|------|------|
| **大通钱庄** | 最高 | 标准利率 | 全国连锁，最安全 |
| **恒泰钱庄** | 高 | 标准利率 | 老字号，信誉好 |
| **利源钱庄** | 中 | 利率+20% | 新兴钱庄，服务灵活 |
| **地下钱庄** | 低 | 利率+50% | 不合法，风险高 |

## 十一、系统联动

### 11.1 金币同步（EconomySystem <-> Player）

经济系统通过 `link_player()` 方法与玩家对象建立双向同步：

```python
# 链接玩家
economy.link_player(player)

# 读取金币时优先从 player.money 获取
@property
def gold(self) -> int:
    if self._player_ref:
        return self._player_ref.money
    return self._gold

# 设置金币时同时更新玩家
@gold.setter
def gold(self, value: int) -> None:
    self._gold = value
    if self._player_ref:
        self._player_ref.money = value
```

### 11.2 声望与经济的联动

声望等级直接影响商店折扣：

```python
def get_shop_discount(self, faction: str) -> float:
    standing = self.get_faction_standing(faction)
    discounts = {
        "掌门亲传": 0.3,
        "门派长老": 0.2,
        "门派弟子": 0.15,
        "外门弟子": 0.1,
        "中立": 0.0,
        "门派仇人": -0.2,
        "不死不休": -0.5,
    }
    return discounts.get(standing, 0.0)
```

## 十三、总结

"白金英雄坛说"的经济与声望系统是一个深度与广度兼备的江湖生态模拟系统。通过货币体系、商店系统、拍卖行、黑市、钱庄、声望系统、正邪阵营等模块的有机结合，为玩家提供了丰富的商业和社交体验。

系统的核心设计理念包括：

1. **真实模拟**：供需关系、季节性波动、事件影响等机制模拟真实经济
2. **道德选择**：正邪阵营系统让玩家面临道德抉择，影响游戏进程
3. **社交互动**：声望系统和NPC关系系统增强了游戏的社交性
4. **策略深度**：套利、投资、拍卖等玩法提供了丰富的策略选择
5. **风险回报**：黑市、贷款等高风险高回报的玩法增加了游戏的刺激性

这一系统不仅为游戏提供了丰富的玩法内容，也为AI开发提供了清晰的接口和数据结构，是"白金英雄坛说"的重要组成部分。
