# 一、战斗系统概述

## 1.1 设计理念：为什么选择混合模式

本游戏采用**混合战斗模式**，将两种截然不同的战斗体验融合在一起：

| 模式 | 说明 | 设计目的 |
|------|------|---------|
| **大地图半即时探索** | 玩家在大地图上自由移动，系统以半即时方式检测遇敌 | 营造开放世界的沉浸感，玩家可以自由探索 |
| **回合制战斗界面** | 进入战斗后切换为经典回合制策略战斗 | 提供策略深度，玩家需要合理选择行动 |

### 混合模式的核心优势

1. **探索的自由感**: 半即时大地图让玩家自由移动，不受回合制限制
2. **战斗的策略感**: 回合制战斗让每一步都经过深思熟虑
3. **节奏的切换感**: 从探索到战斗的切换带来紧张刺激的体验
4. **水墨风格的统一感**: 战斗界面采用水墨风格，与武侠主题完美契合

## 1.2 战斗节奏设计

不同类型的战斗有不同的预期回合数：

| 战斗类型 | 预期回合数 | 设计目标 | 典型敌人 |
|---------|-----------|---------|---------|
| 普通战 | 3-5回合 | 快速解决，保持节奏 | 山贼、野狼、小贼 |
| 精英战 | 8-12回合 | 需要策略思考 | 流寇、叛徒、仇家 |
| Boss战 | 15-20回合 | 高难度挑战，多阶段 | 门派掌门、世界Boss |
| 传说战 | 20-30回合 | 终极挑战，需要充分准备 | 传说Boss |

### 战斗节奏控制机制

- **普通战**: 敌人属性较低，玩家可以快速击败，保持探索的流畅感
- **精英战**: 敌人有技能和特殊机制，需要玩家合理使用药品和技能
- **Boss战**: 多阶段设计，Boss在不同阶段使用不同技能，需要玩家适应
- **传说战**: 极高难度，需要特定的策略和装备搭配

## 1.3 战斗系统核心特色

1. **大地图探索** -- 半即时移动，遇敌检测（`EncounterSystem`）
2. **回合制战斗** -- 策略性回合战斗，需要合理选择行动（`HybridCombatSystem`）
3. **战斗UI** -- 水墨风格战斗界面
4. **战斗动画** -- 招式演出效果（`ANIMATION` 阶段预留）
5. **门派特色** -- 七大门派各有独特武学招式
6. **元素克制** -- 金木水火土五行相克系统
7. **状态效果** -- 中毒/灼烧/冰冻/眩晕/流血/内伤等多种状态

# 二、战斗阶段详解

## 2.1 八大战斗阶段

战斗通过 `CombatPhase` 枚举管理，共 8 个阶段：

| 阶段 | 枚举值 | 说明 | UI变化 | 可执行操作 |
|------|--------|------|--------|-----------|
| 非战斗状态 | `NONE` | 玩家处于大地图探索状态 | 显示大地图UI | 移动、交互、采集 |
| 战斗初始化 | `INIT` | 战斗开始时的初始化阶段 | 战斗界面淡入，显示双方信息 | 无（自动进行） |
| 玩家回合 | `PLAYER_TURN` | 当前行动者为玩家 | 高亮玩家角色，显示行动菜单 | 攻击/技能/物品/防御/逃跑 |
| 敌人回合 | `ENEMY_TURN` | 当前行动者为敌方 | 高亮敌人角色，显示思考动画 | AI自动选择 |
| 播放动画 | `ANIMATION` | 播放招式动画效果 | 全屏动画覆盖 | 无（自动进行） |
| 战斗胜利 | `VICTORY` | 所有敌人被击败 | 显示"战斗胜利"横幅，弹出奖励面板 | 确认奖励、继续探索 |
| 战斗失败 | `DEFEAT` | 玩家HP归零 | 显示"战斗失败"横幅，弹出复活选项 | 复活/返回城镇 |
| 逃跑 | `FLEE` | 玩家成功逃离战斗 | 显示"成功逃脱"提示 | 继续探索 |

## 2.2 阶段转换条件

```
NONE ──(遇敌检测成功)──→ INIT
INIT ──(初始化完成)──→ PLAYER_TURN 或 ENEMY_TURN（根据速度排序）
PLAYER_TURN ──(玩家选择行动并执行)──→ ENEMY_TURN 或 ANIMATION
ENEMY_TURN ──(AI选择行动并执行)──→ PLAYER_TURN 或 ANIMATION
ANIMATION ──(动画播放完成)──→ 下一个行动者的回合
任意阶段 ──(所有敌人HP<=0)──→ VICTORY
任意阶段 ──(玩家HP<=0)──→ DEFEAT
PLAYER_TURN ──(玩家选择逃跑且成功)──→ FLEE
```

## 2.3 各阶段详细说明

### 2.3.1 战斗初始化阶段（INIT）

**触发条件**: 遇敌检测成功

**执行流程**（代码 `start_battle` 方法）：
1. 设置 `phase = CombatPhase.INIT`
2. 记录玩家和敌人信息
3. 合并所有战斗单位到 `all_combatants` 列表
4. 按速度属性降序排列，生成 `turn_order` 行动顺序
5. 重置所有战斗状态（防御、眩晕、buff列表）
6. 添加战斗日志："战斗开始！{玩家名} 遭遇了 {敌人列表}"
7. 根据行动顺序第一个单位设置 `PLAYER_TURN` 或 `ENEMY_TURN`

**UI变化**: 大地图淡出，战斗界面淡入，显示双方角色立绘和HP/MP条

**动画效果**: 水墨风格的"战斗开始"横幅动画

### 2.3.2 玩家回合（PLAYER_TURN）

**触发条件**: 当前行动者为玩家方单位

**可执行操作**（`ActionType` 枚举）：

| 行动 | 枚举值 | 说明 | 消耗 |
|------|--------|------|------|
| 普通攻击 | `ATTACK` | 使用基础攻击 | 0内力 |
| 武学招式 | `SKILL` | 使用已学武学技能 | 视技能而定 |
| 使用物品 | `ITEM` | 使用背包中的药品等物品 | 消耗物品 |
| 防御 | `DEFEND` | 进入防御姿态 | 0内力 |
| 逃跑 | `FLEE` | 尝试逃离战斗 | 0内力 |

**UI变化**: 玩家角色高亮，底部显示行动选择面板

### 2.3.3 敌人回合（ENEMY_TURN）

**触发条件**: 当前行动者为敌方单位

**执行流程**: AI自动选择行动（详见第五章"战斗AI系统"）

**UI变化**: 敌人角色高亮，显示"敌方行动中..."提示

### 2.3.4 战斗胜利阶段（VICTORY）

**触发条件**: 所有敌人 `hp <= 0`

**执行流程**（代码 `_process_action` 方法）：
1. 设置 `phase = CombatPhase.VICTORY`
2. 添加战斗日志："战斗胜利！"
3. 触发 `on_battle_end(True)` 回调
4. 计算并显示战斗奖励

**UI变化**: 显示"战斗胜利"横幅，弹出奖励面板（经验值、金钱、物品掉落）

### 2.3.5 战斗失败阶段（DEFEAT）

**触发条件**: 玩家 `hp <= 0`

**执行流程**：
1. 设置 `phase = CombatPhase.DEFEAT`
2. 添加战斗日志："战斗失败..."
3. 触发 `on_battle_end(False)` 回调
4. 弹出复活选项

**UI变化**: 显示"战斗失败"横幅，画面变为灰白色调

### 2.3.6 逃跑阶段（FLEE）

**触发条件**: 玩家选择逃跑且判定成功

**逃跑成功率公式**:
```
逃跑概率 = 0.3 + (玩家速度 - 敌人平均速度) * 0.01
```

**UI变化**: 显示"成功逃脱！"提示，战斗界面淡出

# 三、伤害计算系统

## 3.1 基础伤害公式（含完整推导）

### 步骤一：计算基础伤害

```
base_damage = actor.attack * skill.damage_multiplier
```

- `actor.attack`: 攻击方的攻击力属性
- `skill.damage_multiplier`: 技能的伤害倍率（普通攻击为1.0）

### 步骤二：计算防御值

```
defense_value = target.defense * (2 if target.is_defending else 1)
```

- 防御姿态下防御值翻倍
- 防御姿态仅持续当前回合，下一回合自动清除

### 步骤三：计算最终伤害

```
raw_damage = base_damage - defense_value * 0.5
final_damage = max(1, int(raw_damage))
```

- 防御值只减免50%（乘以0.5系数）
- 最低伤害为1点（保底机制）

### 步骤四：暴击判定

```
if random.random() < 0.10:  # 10%暴击率
    final_damage = int(final_damage * 1.5)  # 暴击伤害150%
    is_critical = True
```

### 步骤五：命中判定

```
if random.random() < 0.05:  # 5%闪避率
    final_damage = 0
    is_hit = False
```

### 步骤六：应用伤害

```
target.hp = max(0, target.hp - final_damage)
```

### 完整公式汇总

```
最终伤害 = max(1, int(攻击方.attack * 技能.伤害倍率 - 防御方.defense * 防御系数 * 0.5))
  → 暴击判定(10%): 伤害 * 1.5
  → 命中判定(5%闪避): 伤害 = 0
  → 应用: target.hp = max(0, target.hp - 伤害)
```

### 伤害计算示例

假设攻击方攻击力为50，使用八卦掌（倍率1.2），防御方防御力为20：

| 场景 | 计算 | 结果 |
|------|------|------|
| 普通命中 | max(1, 50*1.2 - 20*0.5) = max(1, 60-10) | **50** |
| 防御状态下 | max(1, 50*1.2 - 40*0.5) = max(1, 60-20) | **40** |
| 暴击 | 50 * 1.5 | **75** |
| 防御+暴击 | 40 * 1.5 | **60** |
| 闪避 | -- | **0** |

## 3.4 闪避系统

### 闪避参数

| 参数 | 基础值 | 说明 |
|------|--------|------|
| 基础闪避率 | 5% | `random.random() < 0.05` |
| 完美闪避率 | 1% | 完美闪避后可触发闪避反击 |

### 闪避率修正因素

| 因素 | 修正值 | 说明 |
|------|--------|------|
| 速度差加成 | +(自身速度-对方速度)*0.2% | 速度越高越容易闪避 |
| 装备加成 | +N% | 闪避率装备属性 |
| 凌波微步 | +20% | 逍遥派轻功技能 |
| 八卦套装(4件) | +15% | 套装效果 |
| 逍遥套装(2件) | +5% | 套装效果 |
| 蟒皮甲 | +3% | 防具特效 |
| 闪避率上限 | 60% | 最多60%闪避率 |

### 闪避反击

当触发完美闪避（1%概率）时：
- 完全躲避攻击（伤害=0）
- 立即对攻击者进行一次反击
- 反击伤害 = 攻击力 * 0.5（普通攻击倍率的一半）
- 反击不消耗内力，不触发暴击

## 3.5 格挡系统

### 格挡参数

| 参数 | 基础值 | 说明 |
|------|--------|------|
| 格挡率 | 8% | 基于防御属性计算 |
| 格挡减伤 | 50% | 格挡成功时减免50%伤害 |
| 格挡反击率 | 30% | 格挡成功后30%概率反击 |

### 格挡率公式

```
格挡率 = 8% + 防御力 * 0.05%
格挡率上限 = 40%
```

### 格挡反击

当格挡成功且触发反击时：
- 伤害减免50%
- 立即对攻击者进行一次反击
- 反击伤害 = 攻击力 * 0.3
- 反击不消耗内力

## 3.6 连击系统

### 连击机制

连续攻击同一目标时，伤害递增：

| 连击次数 | 伤害加成 | 说明 |
|---------|---------|------|
| 第1击 | 100% | 基础伤害 |
| 第2击 | 110% | +10% |
| 第3击 | 125% | +25% |
| 第4击 | 145% | +45% |
| 第5击+ | 170% | +70%（上限） |

### 连击中断条件

- 攻击目标改变
- 使用防御行动
- 使用物品
- 被眩晕/冰冻
- 回合结束

## 3.7 状态效果系统

### 状态效果总览

| 状态 | 代码效果 | 效果描述 | 持续时间 | 叠加规则 | 解除方式 |
|------|---------|---------|---------|---------|---------|
| 中毒 | poison | 每回合造成固定伤害 | 3-5回合 | 可叠加(最多3层)，每层增加伤害 | 解毒药品、自然消退 |
| 灼烧 | burn | 每回合造成火焰伤害 | 3回合 | 不可叠加，刷新持续时间 | 冰属性攻击、自然消退 |
| 冰冻 | freeze | 无法行动，跳过回合 | 1-2回合 | 不可叠加 | 火属性攻击、自然消退 |
| 眩晕 | stun | 无法行动，跳过回合 | 1回合 | 不可叠加 | 自然消退 |
| 流血 | bleed | 每回合造成伤害，移动时额外伤害 | 3回合 | 可叠加(最多3层) | 止血药品、自然消退 |
| 内伤 | internal_injury | 内力持续流失，内力恢复降低50% | 5回合 | 可叠加(最多2层) | 休息、特殊药品 |

### 状态效果详细说明

#### 中毒（poison）

```
伤害 = 基础毒伤 * 叠加层数
基础毒伤 = 施加者攻击力 * 0.1
每回合开始时触发伤害
叠加层数上限: 3层
持续时间: 每层3-5回合（独立计算）
```

#### 灼烧（burn）

```
伤害 = 施加者攻击力 * 0.15
每回合开始时触发伤害
不可叠加，重复施加刷新持续时间
持续时间: 3回合
特殊: 灼烧期间冰冻抗性-50%
```

#### 冰冻（freeze）

```
效果: 目标无法行动，跳过当前回合
持续时间: 1-2回合（由施加者攻击力决定）
触发概率: 冰属性技能自带概率（如寒冰掌15%）
特殊: 冰冻期间受到火属性伤害+50%
解除: 受到火属性攻击时立即解除
```

#### 眩晕（stun）

```
效果: 目标无法行动，跳过当前回合
持续时间: 1回合
代码实现: target.is_stunned = True
触发: 技能具有 effect="stun" 且通过 effect_chance 判定
```

#### 流血（bleed）

```
伤害 = 施加者攻击力 * 0.08 * 叠加层数
每回合开始时触发伤害
叠加层数上限: 3层
持续时间: 每层3回合
特殊: 流血期间移动（逃跑）时额外受到攻击力*0.2的伤害
```

#### 内伤（internal_injury）

```
效果: 每回合流失内力 = 最大内力 * 5% * 叠加层数
      内力恢复效果降低50%
叠加层数上限: 2层
持续时间: 每层5回合
特殊: 内伤期间使用内力技能消耗+20%
```

# 四、武学招式大全

## 4.1 八卦门武学

# 五、战斗AI系统

## 5.1 AI决策树

### 基础决策逻辑

当前代码实现了简单随机AI（`enemy_ai_action` 方法），设计文档扩展为智能决策树：

```
┌─────────────────────────────────┐
│         AI决策开始              │
└──────────┬──────────────────────┘
           │
    ┌──────▼──────┐
    │ 检查自身血量 │
    └──────┬──────┘
           │
    ┌──────┼──────────┐
    │      │          │
 ▼ >70%  ▼ 30-70%  ▼ <30%
 进攻型  平衡型    防御型
    │      │          │
    │  ┌───▼───┐     │
    │  │检查内力│     │
    │  └───┬───┘     │
    │      │         │
    │  ┌───┼───┐    │
    │  │>50% │<50%│  │
    │  ▼     ▼    │  │
    │ 技能  普通   │  │
    │ 攻击  攻击   │  │
    │              │
    │         ┌────▼────┐
    │         │使用药品  │
    │         │或防御    │
    │         │或逃跑    │
    │         └─────────┘
```

### 血量分档行为

| 血量范围 | AI类型 | 行为特征 | 技能使用概率 |
|---------|--------|---------|------------|
| >70% | 进攻型 | 优先使用高伤害技能 | 40% |
| 30%-70% | 平衡型 | 根据内力和情况灵活选择 | 30% |
| <30% | 防御型 | 优先使用药品和防御 | 15% |

## 5.2 AI技能使用优先级

### 通用优先级规则

```
1. 自身血量 < 30% 且有药品 → 使用药品
2. 目标血量 < 20% 且有高伤害技能 → 使用高伤害技能（斩杀）
3. 自身有增益buff即将消失 → 刷新buff
4. 目标有debuff → 使用攻击技能（趁虚而入）
5. 内力 > 50% → 使用技能（30%概率选择技能，70%普通攻击）
6. 内力 <= 50% → 普通攻击为主（10%概率选择技能）
```

### 门派AI特殊优先级

| 门派 | 优先使用技能 | 条件 | 备选策略 |
|------|------------|------|---------|
| 八卦门 | 八卦刀法 | 装备刀类武器 | 八卦掌/混元一气 |
| 花间派 | 百花缭乱 | 敌人数量>=2 | 落花剑/暗香掌 |
| 红莲教 | 血刃 | 自身血量<70% | 红莲密咒/义拳 |
| 那迦派 | 影袭 | 目标血量<30% | 忍刀/烟遁(血量<20%) |
| 太极门 | 云手 | 对手准备强力攻击 | 太极拳/太极剑 |
| 雪山派 | 雪崩 | 内力充足且目标血量>50% | 雪山刀/寒冰掌 |
| 逍遥派 | 北冥神功 | 自身内力<30% | 凌波微步(血量<50%)/六阳掌 |

# 十、代码实现对照

## 10.1 核心数据结构

| 设定项 | 文档设定 | 代码实现 | 对应类/枚举 |
|--------|---------|---------|-----------|
| 战斗阶段 | 8个阶段 | `CombatPhase` 枚举 | NONE/INIT/PLAYER_TURN/ENEMY_TURN/ANIMATION/VICTORY/DEFEAT/FLEE |
| 行动类型 | 5种行动 | `ActionType` 枚举 | ATTACK/SKILL/ITEM/DEFEND/FLEE |
| 战斗单位 | 7个基础属性+4个状态属性 | `Combatant` 数据类 | name/max_hp/hp/max_mp/mp/attack/defense/speed |
| 战斗行动 | 8个字段 | `CombatAction` 数据类 | actor/action_type/target/skill_name/damage/mp_cost/is_critical/is_hit |
| 技能数据 | 10个字段 | `SkillData` 数据类 | name/name_zh/description/mp_cost/damage_multiplier/element/target_type/effect/effect_chance/animation |

## 10.2 伤害计算代码对照

| 设定项 | 代码实现 | 代码位置 |
|--------|---------|---------|
| 基础伤害 | `base_damage = actor.attack * skill.damage_multiplier` | `_execute_action` 方法 |
| 防御计算 | `defense = target.defense * (2 if target.is_defending else 1)` | `_execute_action` 方法 |
| 最终伤害 | `damage = max(1, int(base_damage - defense * 0.5))` | `_execute_action` 方法 |
| 暴击判定 | `if random.random() < 0.1: damage = int(damage * 1.5)` | `_execute_action` 方法 |
| 命中判定 | `if random.random() < 0.05: damage = 0` | `_execute_action` 方法 |
| 伤害应用 | `target.hp = max(0, target.hp - damage)` | `_execute_action` 方法 |

## 10.3 特殊效果代码对照

| 效果 | 触发条件 | 代码实现 | 拥有技能 |
|------|---------|---------|---------|
| drain(吸取内力) | `effect="drain"` + `effect_chance` | `target.mp -= drain; player.mp += drain` | 北冥神功 |
| stun(击晕) | `effect="stun"` + `effect_chance` | `target.is_stunned = True` | (预留) |
| flee(逃跑) | `effect="flee"` | 等同于逃跑行动 | 烟遁 |
| dodge(闪避) | `effect="dodge"` | 提升闪避率 | 凌波微步 |

turn_order = sorted(all_combatants, key=lambda c: c.speed, reverse=True)

每轮结束后重新计算行动顺序（仅包含存活单位）。
